title: Custom layouts

MouseTiler provides a powerful layout system that lets you create custom tile configurations beyond the built-in presets. You can define layouts using a simple text-based syntax or use the visual web editor.

## Layout syntax overview

Custom layouts use a coordinate-based syntax where each tile is defined by its position and dimensions:

```
x,y,width,height
```

- **x**: Horizontal position (0-100% of screen width)
- **y**: Vertical position (0-100% of screen height)  
- **width**: Tile width (0-100% of screen width)
- **height**: Tile height (0-100% of screen height)

Multiple tiles are separated by `+` symbols, and you can add optional names and descriptions using `;` separators.

### Basic examples

<CodeGroup>
```text Two equal vertical tiles
0,0,50,100+50,0,50,100
```

```text Three horizontal sections
0,0,100,33+0,33,100,34+0,67,100,33
```

```text Master and stack layout
0,0,67,100+67,0,33,50+67,50,33,50
```
</CodeGroup>

## Grid-based layouts

For simple grid layouts, use the shorthand `WxH` syntax:

- `2x2` creates a 2-column by 2-row grid (4 tiles)
- `3x1` creates a 3-column by 1-row grid (3 tiles)
- `1x4` creates a 1-column by 4-row grid (4 tiles)

MouseTiler automatically calculates tile positions and handles rounding for even distribution.

## Adding layout names

You can add a descriptive name to any layout by appending it with a semicolon:

```text
0,0,75,100+25,0,75,100+25,0,50,100;75 50 75 (%)
```

This layout appears in the tiler interface with the name "75 50 75 (%)".

## Special layouts

MouseTiler includes special layout commands that provide advanced functionality:

    Maximizes the window. It returns to its previous size when moved.
  </Card>
  
  <Card title="SPECIAL_MINIMIZE" icon="window-minimize">

  </Card>
  
  <Card title="SPECIAL_FULLSCREEN" icon="expand">

  </Card>

  </Card>

  </Card>

## Auto-tiler special layouts

MouseTiler includes shortcuts for switching between auto-tiler configurations:

- `SPECIAL_AUTO_TILER_TOGGLE`: Toggle auto-tiling on/off
- `SPECIAL_AUTO_TILER_1`: Switch to auto-tiler configuration 1
- `SPECIAL_AUTO_TILER_2`: Switch to auto-tiler configuration 2  
- `SPECIAL_AUTO_TILER_3`: Switch to auto-tiler configuration 3

## Configuration files

You can configure layouts in two places:

### Popup grid layouts

Edit the `popupLayout` setting to customize which layouts appear in your main popup tiler grid:

```text
1x1
2x1
3x1
SPECIAL_SPLIT_HORIZONTAL;Horizontal Split
0,0,75,100+25,0,75,100+25,0,50,100;75 50 75 (%)
4x1
2x2
SPECIAL_FILL;Fill
4x2
SPECIAL_AUTO_TILER_1
SPECIAL_AUTO_TILER_2
SPECIAL_AUTO_TILER_3
```

### All layouts grid

The `allPopupLayouts` setting contains a comprehensive collection of all available layouts. Access this extended grid with the "Show All" shortcut (default: `Ctrl+Space`).

### Overlay tiler layout

Set the `overlayLayout` configuration to define the tile grid for the full-screen overlay tiler:

```text
4x2
```

## Advanced layout features

### Pixel-based positioning

Instead of percentages, you can use pixel values by adding `px`:

```text
0,0,800px,600px
```

### Fractional values

Use fractions for precise positioning:

```text
0,0,1/3,100+1/3,0,1/3,100+2/3,0,1/3,100
```

This creates three equal columns using fractions instead of percentages.

### Anchor points

For layouts with 6 parameters, the last two define anchor points:

```text
x,y,width,height,anchorX,anchorY
```

Anchor points support named values:
- Horizontal: `left`, `center`, `right`
- Vertical: `top`, `center`, `bottom`

Or use percentage/fractional values (0-100).

## Managing layouts

<Steps>
  <Step title="Edit configuration">
    Open MouseTiler settings and locate the `popupLayout`, `allPopupLayouts`, or `overlayLayout` setting.
  </Step>
  
  <Step title="Add your layout syntax">
    Each line represents one layout. Use the syntax described above or generate it from the web editor.
  </Step>
  
  <Step title="Reload the script">
    Due to a KDE limitation, you must reload MouseTiler after changing settings:
    1. Untick MouseTiler in KWin Scripts settings
    2. Click Apply
    3. Tick MouseTiler
    4. Click Apply
  </Step>
</Steps>

<Info>
The `rememberAllLayouts` setting controls whether MouseTiler remembers which layout you last used in the extended grid.
</Info>

## Tips for creating layouts

1. **Start simple**: Begin with basic percentage-based layouts before adding complexity
2. **Test incrementally**: Add one tile at a time and test to ensure correct positioning
3. **Use the web editor**: For complex multi-tile layouts, the visual editor is much faster
4. **Name your layouts**: Descriptive names make it easier to identify layouts in the grid
5. **Consider margins**: Remember that tile and screen margins (configured separately) will be applied to your layouts

<Tip>
You can combine multiple layout approaches in your configuration. Use grid syntax for simple layouts and coordinate syntax for complex ones.
</Tip>
